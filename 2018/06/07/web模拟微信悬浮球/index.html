web模拟微信悬浮球

看到张鑫旭老前辈发的用黑科技clip-path模拟微信安卓端最近更新的悬浮球效果，顿时激发自己也写一个试试的想法，由于clip-path在兼容方面相当不好，所以这里就用现代浏览器可以兼容的写法来实现。

先贴最终效果：

效果描述：

悬浮球点击后打开的页面，点击左上角 X 关闭页面。
在关闭页面的时候，有一个动画效果：像是把页面缩回了悬浮球里。

实现原理：

悬浮球打开的页面相当于一个独立的webview（可以理解为浏览器的一个标签页），这里直接用一个z-index较高的遮罩层来模拟了。
关键点：点击关闭的时候动画加载，这里用transition过渡动画，改变容器的宽高和圆角属性，同时设置容器超出隐藏效果。

分解步骤：

  1. 简单布局微信黑色header，和悬浮窗打开后的样式

  2. 实现点击小球展开页面效果

    默认有一个悬浮球在微信首页，（这里悬浮球的拖动和边缘吸附网上很多教程，这里就不做了）。其实是一个容器，里面放置一个灰色遮罩层，用于遮挡页面内容，再放置一个层级较高的img图片（就是这个链接图标）。
    点击小球后，

    1. 先让img图片隐藏，
2. 再改变容器的样式，进行动画过渡，
3. 过渡结束后隐藏灰色遮罩层，显示出页面内容。

  3. 实现点击页面 X 收缩页面到悬浮球里

    点击x后：
    1.先显示遮罩层，遮挡页面内容（因为过渡动画的过程中，文档流布局会因为容器宽高变化而实时的发生的回流）
    2.改变容器样式，进行动画过渡，
    3.过渡结束后，显示img图标

tips:

当我们在js中用document.getElement获取元素的时候，可以选择ById/ByClassName/ByTagName，这里唯有ById是最快的

浏览器解析html代码形成DOM树的时候，每个节点的属性都挂在每个节点上，

用ID选择器时候，可以直接从DOM树上拿到对应的一个元素

而用clss或者tag选择器时，引擎会遍历css树（非专有名词），一一匹配来拿到一组元素。

根据页面复杂程度，ID选择器会比后两者选择器性能上快上百倍。
